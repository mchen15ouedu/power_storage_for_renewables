# Solar + Wind Storage Atlas — website

A self-contained **static site** for the GLDAS / NLDAS renewable-storage analysis. An
interactive Leaflet map of all 770 regions (admin0 + admin1); click a region for its
storage requirement, capacity factors, supply-vs-demand charts, co-location results, and —
for USA states — real-units (TWh, nameplate GW, land-use feasibility) driven by real EIA
consumption, with a GLDAS ↔ NLDAS dataset toggle.

## Layout
```
web/
  index.html  css/style.css  js/app.js   # the app (Leaflet + Plotly via CDN)
  CNAME  .nojekyll                        # GitHub Pages config (custom apex domain)
  data/                                   # generated by scripts/build_website.py
    meta.json  regions.geojson  regions_index.json
    regions/<id>.json                     # per-region numbers + 12-month series
    figures/<id>.pdf                      # per-region detailed figures
    downloads/*.csv                       # raw result tables
```

## Regenerate the data
From the repo root, in the `geo2` conda env:
```bash
conda activate geo2
python scripts/build_website.py --scope all --with-pdfs
```
- `--scope usa|world|all` — which regions get per-region charts (numbers + geojson always
  cover all 770). `--with-pdfs` also renders `data/figures/<id>.pdf` (~80–90 MB for 770).
- The builder reads `gldas_analysis/results/*.csv` and `data/regions.gpkg` and reuses the
  analysis package (`gldas_storage.{analyze,deficit,landuse}`) to recompute the series.
- **NLDAS-USA** auto-fills the USA "NLDAS" tab once `nldas_analysis/results/usa_summary.csv`
  exists — just re-run the builder; matched to states by name (CONUS only, no AK/HI).

## Deploy to GitHub Pages → https://mcspace.work/
The site is the **contents of `web/`**. Serve it at the apex domain root (relative paths,
no base-path needed).

1. Create a repo (e.g. `mcspace-site`) whose **root = the contents of this `web/` folder**
   (copy `index.html`, `css/`, `js/`, `data/`, `CNAME`, `.nojekyll`). Commit and push.
   - Large `data/` (geojson 6.8 MB + ~80–90 MB of PDFs): all files are < 100 MB and the
     total is < 1 GB, so plain git is fine (no LFS).
2. Repo → **Settings → Pages**: Source = *Deploy from a branch*, Branch = `main`, folder
   `/ (root)`. Save.
3. Pages → **Custom domain** = `mcspace.work` (the `CNAME` file already sets this). Enable
   *Enforce HTTPS* once the cert is issued.
4. At your DNS registrar for `mcspace.work`:
   - Apex `A` records → GitHub Pages IPs: `185.199.108.153`, `185.199.109.153`,
     `185.199.110.153`, `185.199.111.153` (and the matching `AAAA` records if you want IPv6).
   - `CNAME` for `www` → `<your-github-username>.github.io`.
5. Wait for DNS + the Pages cert; the site comes up at https://mcspace.work/.

## Notes
- Leaflet, Plotly, and the Esri/CARTO basemap tiles load from CDNs (need internet at view
  time — fine on Pages). To run fully offline, vendor those libraries locally.
- Basemaps: Satellite (Esri World Imagery, default), Topographic, Terrain, Dark — switch via
  the control at bottom-left. No API key required.
